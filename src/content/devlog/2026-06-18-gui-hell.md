I really hate GUI programming. In Script Kitties, I've rewritten the GUI engine twice. The first time was when I went from text and boxes to sprites:

![Rewrite #1](/devlog/gui-hell/gui-rewrite-1.png)

And the second was when I realized you shouldn't design GUIs in pure C++.

```cpp
// This was a simple example
Window->AddWidget(RootGUI->NewPanel()
    .Layout(WindowContents)
    .Trim(WidgetTrim::None)
    .AddWidget(RootGUI->NewPanel()
        .Position({ 1, 0, 4, 0 })
        .Size({ 0, 0, 275, 48 })
        .BorderSize(1)
        .BorderColor(FocusedFrameColor)
        .Background({ 0, 0, 0, 255 })
        .AddWidget(RootGUI->NewImage("gui_kittycoin.png")
            .Size({ 0, 0, 48, 48 })
            .AddWidget(RootGUI->NewText()
                .Position({ 1, 0, 5, 18 })
                .Size({ 1, 1, 0, 0 })
                .Update([](WText<ShopState>& Widget, const ShopState& State) {
                    Widget.Text = GameManager::Only().Bag->Money;
                    })
                .Build())
            .Build())
        .Build())
    .Build());
```

_The above code rendered the following, by the way:_

![For this?](/devlog/gui-hell/cpp_gui.png)

It was a tedious process of writing code, compiling, validating the GUI, writing some more code, and compiling again. It took 40 hours to write the battle GUI and the main thing I learned was **I did not want to do this again**. 

![40 Excruciating hours](/devlog/gui-hell/battle-scene.png)

### LGUI

I didn't want to write GUIs in C++ anymore, so I created bindings in C++ to write GUIs in Lua instead (hence, ***L***GUI). It was significantly faster, not just because the language itself was smoother to program in but also because with Lua I could finally do *hot reloading*. I'd make a change and see it reflected in-game within 50ms, no need to recompile.

![Hot reloading](/devlog/gui-hell/hot_reloading_example.mp4)

### XGUI

LGUI was a major step forward and made GUI programming a lot more tolerable but there were two main pain points that made me stop. The first was that it was still tedious: every element - even a text label - required ~6 lines of code. Less code than C++, but still code. And the second was the separation between the rendering logic (what elements to show, where) and the control logic (the player moved the mouse, now what?) was easy to blur, which resulted in unmaintainable spaghetti code.


```lua
-- Same as the C++ code above
local window = GUI:createPanel({
    parent = root,
    position = {1, 0, 4, 0},
    size = {0, 0, 275, 48},
    borderSize = 1,
    borderColor = Theme.FocusedFrameColor,
    backgroundColor = {0, 0, 0, 255}
})
window.sprite = GUI:createPanel({
    parent = window,
    size = {0, 0, 48, 48},
    texture = "gui_kittycoin.png"
})
window.sprite.text = GUI:createText({
    parent = window.sprite,
    position = {1, 0, 5, 18},
    size = {1, 1, 0, 0},
    text = GetBag().money
})
```

So this time I set out to build a full drag-and-drop WYSIWYG visual editor. I settled on XML for representing the view logic and Lua still for the control logic. The pattern I am following is MVC - Model-View-Controller, which is a pretty popular framework for managing complex GUIs. The XML defines a template, populated from a data model, with user-interactivity powered by a Lua controller.
